# 현솔의 사진 콘테스트 — PRD

> Seoul Walking Photo Tour · 사진 미션 콘테스트 앱

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 제품명 | 현솔의 사진 콘테스트 |
| 형태 | 단일 HTML 파일 (서버·앱 설치 불필요) |
| 목적 | 투어 참여자 미션 수행, 교류 촉진, 관리자 현황 파악 |
| 대상 | 서울 산책 투어 참여자 + 운영 관리자 |
| 기술 스택 | HTML / CSS / Vanilla JS, localStorage (메모리 폴백) |

서울과 대한민국의 숨겨진 골목길·명소를 안내하는 산책 투어 가이드 '현솔'이 운영하는 사진 미션 콘테스트 앱입니다.  
참여자들이 투어 중 사진 미션을 수행하고, 서로 친해지며, 상품도 받아가는 참여형 프로그램을 위해 제작되었습니다.

---

## 2. 사용자 유형

### 참여자
- 닉네임 입력만으로 바로 입장
- 5개 미션 수행 (사진 업로드)
- 후기 작성 및 전체 후기 조회

### 관리자
- 비밀번호 6자리 인증: `adgjl7`
- 전체 참여자 미션 완료 현황 확인
- 완료 수 기준 실시간 순위 조회
- 전체 후기 목록 열람

## 3. 미션 구성

| 번호 | 미션명 | 설명 |
|------|--------|------|
| 1 | 우리팀의 단합된 모습 | 팀 전체가 함께하는 사진 |
| 2 | 우리팀의 인물사진 | 팀원 개인 또는 그룹 인물 사진 |
| 3 | 오늘의 색깔을 찍어봐요 | 오늘의 분위기를 색으로 표현 |
| 4 | 사물과 풍경 사진 | 서울 골목의 사물·풍경 포착 |
| 5 | 자연과 동물 사진 | 자연물 또는 동물 사진 |

> 각 미션 사진 업로드 완료 시 **CLEAR** 배지 표시

## 6. 기술 구조

### 스택
- 순수 HTML / CSS / JavaScript — 단일 파일, 외부 서버 불필요
- 데이터 저장: `localStorage` (카카오 인앱브라우저 차단 시 메모리 폴백 자동 적용)
- 사진: `FileReader` API → Base64 → localStorage 저장

### 카카오 인앱브라우저 호환성 처리
- `localStorage` try-catch 래퍼 → 차단 시 메모리 스토어 자동 전환
- `FileReader` onerror 핸들러 → 실패 시 사용자 안내 메시지 표시
- `file input`을 HTML에 미리 고정 선언 (동적 생성 방지)
- `capture="environment"` 속성 → 카메라 직접 실행 지원
- inline `onclick` 제거 → `addEventListener` 방식으로 전환 (CSP 대응)
- 외부 폰트 제거 → 시스템 폰트 사용 (네트워크 차단 환경 대응)

### 데이터 구조 (localStorage key: `hyunsol_v2`)
```json
{
  "users": {
    "닉네임": {
      "missions": [null, "base64_img", null, null, "base64_img"]
    }
  },
  "reviews": [
    { "nickname": "홍길동", "text": "좋았어요!", "time": "3/25 14:30" }
  ]
}
```

## 7. 개발 중 오류 해결 이력

| 오류 | 원인 | 해결 |
|------|------|------|
| `SyntaxError: Unexpected token '}'` | template literal 내부 조건 분기 처리 오류 | 문자열 연결 방식으로 변경 |
| `ReferenceError: loginParticipant is not defined` | inline `onclick` 핸들러가 iframe CSP 정책에 의해 차단 | 모든 inline 이벤트 제거 → `addEventListener` 전면 재작성 |
| `Script error` (카카오 인앱브라우저) | `localStorage` 차단 + `FileReader` 제한 + kakaopixel 스크립트 충돌 | localStorage 안전 래퍼, FileReader try-catch, file input 사전 고정 |

## 8. 운영 가이드

### 시작하기
1. `hyunsol-photo-contest.html` 파일을 카카오톡 / 문자 / QR코드로 공유
2. 참여자: 링크 열기 → 닉네임 입력 → 미션 수행
3. 관리자: 링크 열기 → 관리자 탭 → 비밀번호 `adgjl7` 입력

### 주의사항
- 데이터는 각 기기의 브라우저에 저장됨 (기기 간 동기화 없음)
- 카카오 인앱브라우저보다 **기본 브라우저(사파리/크롬) 사용 권장**
- 브라우저 캐시 삭제 시 저장된 데이터 초기화 가능

### 향후 확장 옵션
- Firebase Realtime DB 연동 → 여러 기기 실시간 동기화
- Netlify / Vercel 배포 → 고정 URL로 공유
- 사진 갤러리 페이지 추가 → 업로드된 미션 사진 모아보기
- 투표 기능 추가 → 참여자 간 베스트 사진 선정
